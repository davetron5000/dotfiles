*java.rmi.server.Skeleton_Description*

The Skeleton interface is used solely by the RMI implementation. 

Every version 1.1 (and version 1.1 compatible skeletons generated in 1.2 using 
rmic -vcompat) skeleton class generated by the rmic stub compiler implements 
this interface. A skeleton for a remote object is a server-side entity that 
dispatches calls to the actual remote object implementation. 


*java.rmi.server.Skeleton.dispatch(Remote,RemoteCall,int,long)*

public void dispatch(

  long hash)

Unmarshals arguments, calls the actual remote object implementation, and 
marshals the return value or any exception.

    theCall - object representing remote call 
    opnum - operation number 
    hash - stub/skeleton interface hash
