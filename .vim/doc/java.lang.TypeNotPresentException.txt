*java.lang.TypeNotPresentException_Description*

Thrown when an application tries to access a type using a string representing 
the type's name, but no definition for the type with the specified name can be 
found. This exception differs from (|java.lang.ClassNotFoundException|) in that 
ClassNotFoundException is a checked exception, whereas this exception is 
unchecked. 

Note that this exception may be used when undefined type variables are accessed 
as well as when types (e.g., classes, interfaces or annotation types) are 
loaded.
